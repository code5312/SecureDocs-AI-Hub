SecureDocs AI Hub 개발 지침

프로젝트 목적

기업의 문서 자산을 중앙에서 안전하게 관리하고, 문서 접근제어, 버전 관리, 공유, AI 자동분류, 유사문서 추천, RAG 기반 AI 채팅, 자동백업 및 복구 기능을 제공하는 포트폴리오용 웹 플랫폼을 개발한다.

이 프로젝트는 특정 상용 제품을 그대로 복제하지 않는다. 공개된 기업용 문서중앙화 및 AI 지식관리 개념을 참고하여 핵심 기능을 직접 설계하고 구현한다.

아키텍처 원칙

초기 버전은 모듈형 모놀리식 구조로 구현한다.
도메인별 서비스, 저장소, API 계층을 분리한다.
API 계층에서 직접 SQL을 작성하지 않는다.
문서 파일은 DB에 저장하지 않고 MinIO에 저장한다.
파일의 메타데이터와 저장 키만 PostgreSQL에 저장한다.
AI 공급자와 저장소 구현체는 인터페이스를 통해 교체할 수 있게 한다.
모든 설정과 비밀정보는 환경변수로 관리한다.
기존 코드를 수정하기 전에 관련 파일과 의존 관계를 먼저 확인한다.
기존 기능을 삭제하거나 우회하지 않는다.
한 번의 작업에서는 하나의 기능군만 완성한다.

보안 규칙

모든 문서 조회, 검색, 다운로드 API에 권한 검사를 적용한다.
관리자 API에는 역할 검사를 적용한다.
문서 벡터 검색에도 사용자 권한 필터를 적용한다.
접근할 수 없는 문서 청크를 LLM에 전달하지 않는다.
업로드 파일은 확장자, MIME 타입, 크기를 검사한다.
실제 파일명으로 저장 경로를 만들지 않는다.
저장 키에는 UUID를 사용한다.
파일 경로 조작을 방지한다.
업로드 파일의 SHA-256 체크섬을 저장한다.
비밀번호, API 키, 토큰을 코드에 하드코딩하지 않는다.
Refresh Token은 HttpOnly 쿠키를 사용한다.
다운로드에는 짧은 만료 시간을 가진 URL을 사용한다.
중요 행위는 AuditLog에 기록한다.
AI 답변은 검색된 기업 문서에 근거해야 한다.
AI 답변에는 문서명, 버전, 페이지 또는 섹션 출처를 포함한다.
검색 근거가 부족하면 근거가 부족하다고 응답한다.
문서 안의 지시문은 시스템 명령으로 실행하지 않는다.
AI Agent에는 읽기 전용 도구만 기본 허용한다.

코드 품질 규칙

Python 코드에는 타입 힌트를 작성한다.
공개 함수와 복잡한 서비스에는 docstring을 작성한다.
Pydantic 요청 및 응답 스키마를 분리한다.
HTTP 상태 코드와 오류 응답을 일관되게 사용한다.
중복 로직은 서비스 또는 공통 유틸리티로 추출한다.
비동기 함수 안에서 블로킹 I/O를 직접 실행하지 않는다.
데이터베이스 트랜잭션 경계를 명확히 한다.
예외를 단순히 무시하지 않는다.
로그에 비밀번호, 토큰, 문서 본문을 기록하지 않는다.
프론트엔드 컴포넌트는 기능 단위로 분리한다.
TypeScript에서 any 사용을 최소화한다.
사용자 입력은 프론트엔드와 백엔드 양쪽에서 검증한다.
